正则表达式简单入门

发表于 2017-07-26 
## 正则表达式提供了一种强大的文本搜索和处理方式,缩写为regex或者regexp

“正则表达式”描述在搜索文本正文时要匹配的一个或多个字符串。 该表达式可用作
一个将字符模式与要搜索的字符串相匹配的模板。正则表达式包括普通字符（例如，a 到 z 之间的字母）
和特殊字符（称为“元字符”）。

** 在javascript中有两种方式 **
一种是用内建构造器RegExp()来创建正则表达式对象
例如：var a = new RegExp("j.*t");
一种是更为简便的正则文本标记法：var a = /j.*t/;
以上示例中  j.*t 就是正则表达式模式,该含义就是"匹配任何以j开头以t结尾的字符串,
这两个字符之间可以包含一个或者多个字符"，*意思就是"一个或者多个单元"，
.代表任意字符。
**注意在向RegExp构造器传递模式时，必须放在一对引号中**

## RegExp对象的属性
**正则表达式拥有五个属性**
global:只读属性，不填写默认值为false，相关搜索在匹配到第一个就会停止搜索，如果要找出所有匹配的就将其值设置为true(填写它的简写 g)。
ignoreCase:只读属性，设置大小写相关性，默认为false(填写它的简写 i 就修改为true)
multiline:只读属性，设置是否跨行搜索，默认值围false(填写它的简写 m 就修改为true)
lastIndex:返回字符串中下一个匹配串的开始位置，该方法只有在test()和exec()成功匹配之后，并且global修饰符被设定时才有效
source:只读属性，返回该正则表达式的模式(不包含修饰符)
**注意除了lastIndex外，上面所有属性在对象创建后就不能在修改**
## 使用方法：
var a = new RegExp("j.*t","gim");
var a = /j.*t/gim;

## Regexp对象的方法
有两种可用于查找匹配内容的方法 test()和exec()
这两种方法参数都是一个字符串，test()返回的是布尔值(找到时为true,否则为false)，
exec()返回的是由匹配到的字符串组成的数组。
例： test()  /j.*t/.test("Javascript") 返回false
    /j.*t/i.test("Javascript") 返回true 因为添加了忽视大小写的的i修饰符
    exec() /j.*t/.exec("Javascript") 访问它返回数组的首元素  结果"Javascript"

## 以正则表达式为参数的方法
String对象为我们提供了这些方法，在String中，以正则表达式对象围参数的方法有以下几种：
var a = new String("OxfordUniversityPress")

match():返回一个包含匹配内容的数组
a.match(/s/); ["s"]  a.match(/s/g); ["s","s","s"]   a.match(/o.*u/gi); ["OxfordU"]

search():返回第一个匹配内容所在守字符的索引位置
a.search(/s.*y/g);  12

replace():将相关的匹配文本替换成某些其它字符串
将所有的大写字母替换成空字符串
var a = new String("OxfordUniversityPress"); a.replace(/[A-Z]/g," ");
" xford niversity ress"
将所有大写字母替换成下划线
var a = new String("OxfordUniversityPress"); a.replace(/[A-Z]/g,"_");
"_xford_niversity_ress"
给所有大写字幕前加下划线
var a = new String("OxfordUniversityPress"); a.replace(/[A-Z]/g,"_$&");
"_Oxford_University_Press"

## 正则表达式的匹配模式
1.[abc]  匹配字符类信息
 I love you".match(/[oyu]/g);
 ["o", "y", "o", "u"]

2.[a-z] 匹配某以恶区间内的字符信息，例如[a-d]相当于[abcd],[a-z]表示匹配所有小写字母
[a-zA-Z]匹配所有的大小写字母，[a-zA-Z0-9_]匹配所有的大小写字母 数字 以及下划线
 "I love you".match(/[a-z]/g);
 ["l", "o", "v", "e", "y", "o", "u"]
 "I love you".match(/[a-zA-Z]/g);
 ["I", "l", "o", "v", "e", "y", "o", "u"]

 3.[^abc] 匹配所有不属于表达式限定范围内的字符
 _Oxford_University_Press".match(/[^a-z]/g);
 ["_", "O", "_", "U", "_", "P"]

 4.a|b 匹配a或b |或者的意思，该符号可在同一表达式中多次使用

 5.a(?=b) 匹配所有后面跟着b的信息
    a(?!b) 匹配术语后面不跟着b的信息

 6.\ 帮助我们匹配一些模式文本中的特殊字符
 "R2-D2".match(/[2-3]/g);
  ["2", "2"]
  "R2-D2".match(/[2\-3]/g);
  ["2", "-", "2"]

  7.\n 换行符  \r 回车符  \f 换页符 \t 横向制表符 \v 纵向制表符

  8.\s 匹配空白符，页包括上面五个转义字符

  9.\S 匹配除空白符以外的所有内容 相当于[^\s]

  10.\w 匹配所有的字母 下划线 数字 相当于[A-Za-z0-9]

  11.\W 匹配的与\w相反

  12.\d 匹配所有的数字信息 相当于[0-9]

  13.\D 与\d相反匹配非数字信息 相当于[^0-9] 或 [^/d]

  14.\0 匹配null值

  15.^匹配字符串开头部分 如果设置了多行修饰就是匹配每一行的开头

  16.$ 匹配输入消息的末尾部分  如果设置了多行修饰就是匹配每一行的末尾

  17. . 匹配除了新行符和换行符以外的任何字符

  18. * 匹配的是模式中间出现0次或者多次的内容，例如/.*/可以匹配任何内容(包括空串)

  19. ？ 匹配的是模式中间出现0次或者1次的内容

  20. + 匹配的是模式中间出现1次或者多次的内容

  21. {n} 匹配的是模式中间出现n次的内容

  22. {min,max} 匹配的是在模式中出现次数min到max之间的信息，如果省略了max 就意味着
  没有最多次数，只有最少次数。但是min不能省略

  23.(pattern) 当某个皮欸模式被放在括号的时候，就表明匹配该模式的匹配串是可替换的，
  因此它也被称为捕获模式，这些捕获的匹配串可以分别用$1,$2,$3...$9等参数来表示
  例如将匹配串中所有的c都重复两次
  "copyfing any classified".replace(/(c)/g,"$1$1$1");
  "cccopying any ccclassified"

 或者将所有的 fi 替换成 if
 "copyfing any classified".replace(/(f)(i)/g,"$2$1");
 "copyifng any classiifed"
